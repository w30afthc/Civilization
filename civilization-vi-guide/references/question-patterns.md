# Question Patterns

Use these templates to answer quickly and consistently.

## Situation Diagnosis

Ask for missing information only when it changes the answer. Otherwise state assumptions.

First identify the rule set:

- Default to standard rules/base game for this user's normal gameplay.
- If the user says 标准规则/本体/base game, use standard rules.
- If the user says 迭起兴衰/Rise and Fall, use Rise and Fall.
- If the user says 风云变幻/Gathering Storm/全 DLC, use Gathering Storm.
- If the user does not say, do not ask about DLC for ordinary gameplay questions. State "按标准规则" when useful.
- If the user asks about DLC-only systems under standard rules, explain that the system is absent before giving DLC context.

Recommended structure:

```text
我先按标准规则来判断；如果你之后买了资料片，这里可能会变。

结论：你现在最该做的是 [action]，因为 [main reason]。

优先级：
1. [immediate safety or tempo move]
2. [economy/yield move]
3. [victory-route move]

接下来 10 回合：
1. [specific build/move]
2. [specific research/civic/policy]
3. [specific scout/diplomacy/war action]

注意：如果你开了 [DLC/mod/rule], 这里可能要改成 [variant].
```

## Screenshot or Save-State Coaching

If the user provides a screenshot, inspect:

- Turn, era, difficulty, game speed, leader/civilization.
- Rule set/DLC indicators: loyalty/governors/era score for Rise and Fall; disasters, power, climate, diplomatic favor, World Congress, grievances, and Gathering Storm strategic resources for Gathering Storm.
- Top-bar yields: science, culture, gold, faith, strategic resources, amenities/housing if visible.
- City build queues and district options.
- Visible terrain: fresh water, mountains, reefs, resources, chops, coast, chokepoints.
- Military threats: barbarians, enemy units, city defense, walls.
- Victory panel indicators if shown.

Then give:

- Best immediate move.
- One or two alternatives.
- What information would change the recommendation.

## Mechanic Explanation

Use this shape:

```text
[术语] 可以理解成 [plain-language metaphor].
它影响 [systems].
实战上你要看三件事：
1. [condition]
2. [timing]
3. [tradeoff]
```

Keep formulas short. If exact numbers are important, browse Civilopedia or an up-to-date rules reference.

## Build Order Advice

Never present a universal build order as always correct. Use "default unless map pressure says otherwise".

Common early baseline:

- Scout if safe and map rewards exploration.
- Slinger/warrior if barbarians or aggressive neighbor pressure.
- Settler once the capital can spare tempo and has protection.
- Builder when there are high-value improvements or boost triggers.
- Monument/granary only when culture/housing is the bottleneck.
- Early district when it supports the chosen path and the city can afford the production.

## District Placement Answer

Evaluate:

- Adjacency value.
- Whether the tile has a strong yield that would be lost.
- Whether the district supports the victory plan.
- Future district/wonder/aqueduct/dam/canal triangles.
- Builder charges/chops needed.
- Housing, amenities, and loyalty constraints.

Output a short ranked list of candidate tiles if the user gives coordinates or a screenshot.

## War Advice

Cover:

- Timing window and target.
- Unit composition.
- Siege plan for walls.
- Upgrade gold and strategic resources.
- Loyalty and occupation stabilization.
- Diplomatic consequences.

If the user is a beginner, remind them that capturing cities without a loyalty/economy plan can stall the empire.

## Tone

Be direct but encouraging. Civilization VI has many hidden systems; explain just enough to make the next decision feel clear.
